# Unity-Gain Sallen-Key Low-Pass Filter

![Rendered Sallen-Key low-pass schematic](preview.svg)

This example demonstrates a unity-gain Sallen-Key low-pass topology using the
repo-owned `CLAW_IDEAL_OPAMP` macromodel. Equal `10 kOhm` resistors and `10 nF`
capacitors place the idealized pole frequency near `1.6 kHz`.

Expected behavior:

- A `1 kHz` sine wave passes with moderate attenuation and phase lag.
- Components are routed explicitly so the rendered `.asc` resembles a readable
  analog filter schematic rather than a cloud of labels.
- The documentation includes both a time-domain response preview and an FFT
  preview showing that high-frequency content is reduced by the filter.

Commands:

```bash
./claw-spice code build examples/transient/sallen-key-lowpass/sallen_key_lowpass.py
./claw-spice sim run examples/transient/sallen-key-lowpass/sallen_key_lowpass.cir
./claw-spice raw plot runs/latest/sallen_key_lowpass.raw V(in) V(filt) V(out) --output runs/latest/sallen_key_lowpass.svg
./claw-spice raw fft runs/latest/sallen_key_lowpass.raw V(out) --output runs/latest/sallen_key_lowpass_fft.svg
./claw-spice show examples/transient/sallen-key-lowpass/sallen_key_lowpass.asc
```

The schematic preview is generated by `ltspice_to_svg`; missing symbols or
flag-only output should be treated as a rendering failure, not accepted as a
valid preview.
