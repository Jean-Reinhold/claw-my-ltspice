# Practical Op-Amp Integrator

![Rendered practical integrator schematic](preview.svg)

This example demonstrates a practical inverting integrator using the repo-owned
`CLAW_IDEAL_OPAMP` macromodel. The input resistor and feedback capacitor form the
integrating path, while the `1 MegOhm` leakage resistor gives the summing node a
DC feedback path so the operating point is well-defined.

Expected behavior:

- A finite-edge `500 Hz` square wave is converted into a small, bounded ramp-like
  output over the measured window.
- The feedback leakage resistor limits DC gain and prevents the ideal integrator
  from drifting indefinitely.
- The committed source netlist includes `../../lib/claw_opamps.lib`; generated
  builds copy the op-amp library beside the output files and include
  `claw_opamps.lib`.

Commands:

```bash
./claw-spice code build examples/transient/opamp-practical-integrator/opamp_practical_integrator.py
./claw-spice sim run examples/transient/opamp-practical-integrator/opamp_practical_integrator.cir
./claw-spice raw plot runs/latest/opamp_practical_integrator.raw V(in) V(out) --output runs/latest/opamp_practical_integrator.svg
./claw-spice show examples/transient/opamp-practical-integrator/opamp_practical_integrator.asc
```

The `.meas` directives report `vout_max`, `vout_min`, and `vout_avg` after the
first few cycles so startup bias does not dominate the measurements.

`preview.svg` is generated by the real `ltspice_to_svg` renderer when you run
`./claw-spice examples render` or `./claw-spice docs assets`.
