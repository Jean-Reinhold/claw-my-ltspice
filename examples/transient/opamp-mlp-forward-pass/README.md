# Op-Amp MLP Forward Pass

![Rendered op-amp MLP forward-pass schematic](preview.svg)

This example implements a small multilayer perceptron forward pass with op-amp
weighted-sum blocks. It uses two input voltages, two hidden neurons in layer 1,
two hidden neurons in layer 2, and one output neuron in layer 3.

The op-amps implement the affine weighted sums. Ideal behavioral voltage sources
model the ReLU activation transfer and signal inversions needed to feed positive
weights into inverting summing amplifiers.

Network equations:

```text
z1a = 0.7*x1 + 0.3*x2 + 0.1
z1b = 0.2*x1 + 0.8*x2 - 0.2
a1a = relu(z1a)
a1b = relu(z1b)
z2a = 0.6*a1a + 0.4*a1b + 0.05
z2b = 0.3*a1a + 0.7*a1b
a2a = relu(z2a)
a2b = relu(z2b)
yout = 0.5*a2a + 0.5*a2b + 0.1
```

Expected behavior:

- Four input vectors are presented as piecewise-constant voltages over an `8 ms`
  transient run.
- The first vector intentionally drives `z1b` negative so `a1b` demonstrates the
  ReLU clamp at `0 V`.
- The output layer produces one measured forward-pass value for each input
  vector.

Commands:

```bash
./claw-spice code build examples/transient/opamp-mlp-forward-pass/opamp_mlp_forward_pass.py
./claw-spice sim run examples/transient/opamp-mlp-forward-pass/opamp_mlp_forward_pass.cir
./claw-spice log summary runs/latest/opamp_mlp_forward_pass.log
./claw-spice raw plot runs/latest/opamp_mlp_forward_pass.raw V(x1) V(x2) V(yout) --output runs/latest/opamp_mlp_forward_pass.svg
./claw-spice show examples/transient/opamp-mlp-forward-pass/opamp_mlp_forward_pass.asc --terminal
```

Key measurements: `y_vec1`, `y_vec2`, `y_vec3`, `y_vec4`, `err_vec1`, `err_vec2`,
`err_vec3`, `err_vec4`, `z1b_vec1`, and `a1b_vec1`.

`./claw-spice examples render` or `./claw-spice docs assets`.
