# Unity-Gain Sallen-Key High-Pass Filter

![Rendered Sallen-Key high-pass schematic](preview.svg)

This example demonstrates a unity-gain Sallen-Key high-pass topology using the
repo-owned `CLAW_IDEAL_OPAMP` macromodel. Equal `10 kOhm` resistors and `10 nF`
capacitors place the idealized pole frequency near `1.6 kHz`, matching the
companion low-pass values while swapping the resistor and capacitor roles.

Expected behavior:

- A finite-edge `1 kHz` pulse train produces edge-emphasized output pulses while
  rejecting the DC portion of the waveform.
- The op amp is used as a unity-gain buffer, with explicit feedback wiring from
  output to the inverting input.
- The committed source netlist includes `../../lib/claw_opamps.lib`; generated
  builds copy the op-amp library beside the output files and include
  `claw_opamps.lib`.

Commands:

```bash
./claw-spice code build examples/transient/sallen-key-highpass/sallen_key_highpass.py
./claw-spice sim run examples/transient/sallen-key-highpass/sallen_key_highpass.cir
./claw-spice raw plot runs/latest/sallen_key_highpass.raw V(in) V(filt) V(out) --output runs/latest/sallen_key_highpass.svg
./claw-spice show examples/transient/sallen-key-highpass/sallen_key_highpass.asc
```

The `.meas` directives report `vout_max`, `vout_min`, and `vout_avg` after the
first millisecond so startup behavior does not dominate the high-pass response.

`preview.svg` is generated by the real `ltspice_to_svg` renderer when you run
`./claw-spice examples render` or `./claw-spice docs assets`.
