# Passive RC Spectrum Split

![Rendered passive RC spectrum split schematic](preview.svg)

This example uses only passive RC branches to split a two-tone source into low
and high frequency outputs. Two series voltage sources create `250 Hz` and
`4 kHz` components at `V(in)`. A `3.3 kOhm`/`47 nF` low-pass branch feeds
`V(low)`, while a matching high-pass branch feeds `V(high)`. Both corners are
near `1 kHz`.

Expected behavior:

- `V(low)` keeps most of the `250 Hz` component and attenuates the `4 kHz`
  component.
- `V(high)` keeps most of the `4 kHz` component and attenuates the `250 Hz`
  component.
- FFT plots of the two outputs should show complementary spectral emphasis
  without active devices or vendor models.

Commands:

```bash
./claw-spice code build examples/transient/passive-rc-spectrum-split/passive_rc_spectrum_split.py
./claw-spice sim run examples/transient/passive-rc-spectrum-split/passive_rc_spectrum_split.cir
./claw-spice raw plot runs/latest/passive_rc_spectrum_split.raw V(in) V(low) V(high) --output runs/latest/passive_rc_spectrum_split.svg
./claw-spice raw fft runs/latest/passive_rc_spectrum_split.raw V(low) --output runs/latest/passive_rc_spectrum_split_low_fft.svg
./claw-spice raw fft runs/latest/passive_rc_spectrum_split.raw V(high) --output runs/latest/passive_rc_spectrum_split_high_fft.svg
./claw-spice show examples/transient/passive-rc-spectrum-split/passive_rc_spectrum_split.asc
```

Measurements:

- `vin_rms`: RMS input voltage over the settled window.
- `low_rms`: RMS low-pass output voltage over the settled window.
- `high_rms`: RMS high-pass output voltage over the settled window.

`preview.svg` is generated by the real `ltspice_to_svg` renderer when you run
`./claw-spice examples render` or `./claw-spice docs assets`.
